# Copilot Instructions for Async_Scraper

## Project Overview

Async_Scraper is a high-performance, async-first web scraping framework designed to turn company names or domains into validated email leads. Built with Python 3.11+ and modern async patterns, it emphasizes performance, reliability, and maintainability.

### Key Features
- **Async-first architecture** using asyncio for high concurrency
- **Multi-layer caching** (L1/L2/L3) for performance optimization
- **Adaptive rate limiting** to respect server limits and avoid blocking
- **Modular design** with clean separation of concerns
- **Comprehensive configuration** management with Pydantic
- **Structured logging** with contextual information
- **CLI interface** with progress tracking
- **Robust testing** with pytest and high coverage requirements

## Code Patterns and Conventions

### Async Patterns
- **Always use async/await** for I/O operations
- **Use aiohttp** for HTTP requests, not requests library
- **Implement proper connection pooling** and resource cleanup
- **Use asyncio.gather() with limits** for concurrent operations
- **Handle asyncio.CancelledError** appropriately

### Error Handling
- Use **tenacity** for retry logic with exponential backoff
- Implement **circuit breaker** patterns for external services
- Always **log errors with context** using structlog
- **Graceful degradation** when services are unavailable

### Configuration Management
- Use **Pydantic models** for all configuration
- Support **environment variables** with `SCRAPER_` prefix
- **JSON and .env file** configuration options
- **Validation and type checking** built into config models

### Caching Strategy
- **L1 Cache**: In-memory LRU cache for frequently accessed data
- **L2 Cache**: Redis cache for shared data (optional)
- **L3 Cache**: File-based cache for persistent data
- **Cache invalidation** strategies based on data freshness

## Important Implementation Details

### Concurrency and Rate Limiting
- Respect **robots.txt** by default
- Implement **adaptive rate limiting** based on response times
- Use **semaphores** to limit concurrent connections per domain
- **Global rate limiting** to prevent overwhelming targets

### Data Processing
- **Normalize column names** when loading CSV/Excel files
- **Validate email formats** using regex and domain validation
- **Remove duplicates** at multiple stages of the pipeline
- **Type coercion and cleaning** for inconsistent input data

### Web Scraping Best Practices
- **User-Agent rotation** and header management
- **Playwright integration** for JavaScript-heavy sites
- **BeautifulSoup** for HTML parsing and email extraction
- **Timeout management** with configurable limits

## Common Tasks and Patterns

### Adding New Configuration Options
1. Add field to relevant Pydantic model in `scraper/core/config.py`
2. Update default values and validation rules
3. Add environment variable mapping if needed
4. Update tests in `tests/unit/test_core.py`

### Adding New Scrapers
1. Create service class in `scraper/services/`
2. Implement async methods with proper error handling
3. Add rate limiting and caching decorators
4. Include comprehensive tests with mocked HTTP responses

### Extending Cache Functionality
1. Add new cache type in `scraper/core/cache/`
2. Implement CacheProtocol interface
3. Update CacheManager to support new type
4. Add configuration options and tests

## Debugging and Troubleshooting

### Common Issues
- **Rate limiting**: Check server response codes and adjust limits
- **Memory usage**: Monitor async task creation and cleanup
- **Cache misses**: Verify cache configuration and key generation
- **Connection timeouts**: Adjust timeout values in configuration

### Logging
- Use **structured logging** with context
- **Log levels**: DEBUG for development, INFO for production
- **Error context**: Include relevant data with error messages
- **Performance metrics**: Log timing and resource usage

This framework is designed for scalability and maintainability. When making changes, always consider the async nature of the codebase and maintain the clean separation of concerns between layers.
