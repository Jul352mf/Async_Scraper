# Copilot Development Configuration

This repository uses GitHub Copilot and this file provides context for better AI assistance.

## Quick Development Setup

```bash
# Install dependencies
pip install -e ".[dev]"

# Run tests
pytest tests/ --cov=scraper

# Format code (required before commits)
black scraper/ tests/

# Lint code
ruff check --fix scraper/ tests/

# Type check
mypy scraper/
```

## Common Development Tasks

### Adding New Features
1. Create feature branch: `git checkout -b feature/new-feature`
2. Add tests first: `tests/unit/test_new_feature.py`
3. Implement feature with proper async patterns
4. Run full test suite and ensure coverage
5. Format and lint code
6. Create pull request

### Configuration Changes  
- Update `scraper/core/config.py` with new Pydantic models
- Add environment variable support where needed
- Update tests and documentation
- Test with `async-scraper test-config`

### Adding New Scrapers
- Create in `scraper/services/`
- Inherit from base classes if available
- Implement async methods with proper error handling  
- Add rate limiting and caching
- Create comprehensive tests with mocked HTTP responses

## Code Quality Checklist
- [ ] All functions have type hints
- [ ] Async/await used for I/O operations
- [ ] Proper error handling with logging
- [ ] Tests written and passing
- [ ] Code formatted with black
- [ ] Linting passes with ruff
- [ ] Type checking passes with mypy
- [ ] Coverage maintained above 25%

## Architecture Notes
- **Async-first**: All I/O operations use async/await
- **Modular**: Clear separation between core, services, data, and CLI
- **Configurable**: Pydantic models for all configuration  
- **Observable**: Structured logging throughout
- **Testable**: Comprehensive test coverage with mocking
